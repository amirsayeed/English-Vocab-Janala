Write the difference between var, let, and const
"Var" is function scoped meaning it is accessible throughout the function it is defined and used to store values, can be reassigned and redeclared whereas "let" and "const" are limited to the block where they are declared. Further they can't be redeclared. However "let" allows reassignment but a variable once declared as "const" it's data can't be changed but in the case of array or object declared as "const" can be modified. A variable declared as "var" is hoisted to the top of their scope making it accessible before declaration but shows undefined. On the contrary "let" and "const" are hoisted but returns reference error if accessed before being declared.
Write The difference between map(), forEach() and filter().
map(), forEach() and filter() each of the method is used to loop through each element in a array and based on condition can return a new array or not. map() performs operation on each element and then returns a new array such as doubling the elements of an array and return the new array. forEach() also excute a function on each element of the array but always return undefined. filter() performs an operation on each elements of the array to filter out the elements that pass the condition and returns them in a new array.
Explain arrow functions and how they are different from regular functions.

Regular Function:

function add(a,b){
return a+b;
}

Arrow Function:

const add = (a,b) => a+b;

How arrow function is different from regular function is given below:

  • Arrow function have implicit return incase of single line expression whereas regular function always need an explicit return.
  • Arrow function can't be used as constructors whereas regular function can be.
  • Arrow function doesn't have arguments object which regular function have
  • Arrow functions don't create their own "this" binding but inherit it from the parent scope, whereas regular functions define their own "this" value based on how they're called.
How JavaScript Promises work?

Promises in Javascript are used to handle asynchronous operation by giving a value which can be available later or not. A promise maybe resolved or rejected with three states such as pending, fullfilled or rejected. A promise being resolved triggers .then() for success and .catch() is triggered for rejected.


const myLoader = () =>{
return new Promise((resolve,reject) => {
const success = Math.random();
if(success > 0.5){
resolve(success);
}else{
reject(success);
}
})
}
myLoader().then(data=>console.log('resolved data',data))
.catch(err => console.log('rejected data',err));
How closures work in JavaScript?

Closures in JS refers to the term when a function is defined within a function but it has access to the variable of the outer function's scope even after outer function's scope has been executed.

function kitchen(){
let roaster = 0;
return function(){
roaster++;
return roaster;
}
}
const firstServer = kitchen();
console.log(firstServer());
console.log(firstServer());
console.log(firstServer());

Here firstServer is assigned the outer function kitchen() which has a roaster variable and returns a function which has access to roaster variable, increments the roaster variable and returns it. The output will be 1, 2 and 3 as the inner function remembers the roaster variable each time and increments it.
